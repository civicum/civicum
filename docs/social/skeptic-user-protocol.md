# Protocolo del Usuario Escéptico (Squad G2)

**Squad:** 35. Escuadrón G2: El Usuario Escéptico  
**Misión:** Simular al usuario que NO cree en nada. Convencerlo.  
**Responsables:** Honestidad Radical, Riesgos Reputacionales, Chilenización, UX Researcher, Fact-checker.

## 1. El Perfil "No te creo nada"
Asumimos que el usuario promedio piensa: "Esto es una estafa", "Van a vender mis datos", o "Es una movida política".
- **Estrategia:** No pedir confianza, *demostrar* confiabilidad.

## 2. Evidencia Verificable (Trustless Design)
- **Código Abierto:** En el footer, siempre un link a "Ver código fuente de esta página".
- **Traza de Dinero:** Link directo a "Quién financia esto" en el menú principal.
- **Blockchain/Log Público:** Si decimos que un voto se contó, mostramos el hash de la transacción (o el log inmutable).

## 3. Lenguaje Desarmante
Evitamos el lenguaje de marketing y el de político.
- **Mal:** "¡Únete a la revolución cívica!" (Suena a estafa).
- **Bien:** "Herramienta gratuita para reportar baches. Sin anuncios, sin venta de datos."

## 4. Manejo de Errores con Humildad
Si la app falla o nos equivocamos:
- **Prohibido:** Mensajes genéricos "Ocurrió un error".
- **Obligatorio:** "Nos equivocamos. El servidor falló porque X. Estamos arreglándolo. Aquí está el status técnico."

*La confianza no se pide, se construye ladrillo a ladrillo con verdad.*
