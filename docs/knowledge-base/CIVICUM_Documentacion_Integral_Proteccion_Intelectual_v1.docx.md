# **1\. IDENTIDAD CIVICUM**

## **1.1 Definición**

* **Nombre:** CIVICUM \- Sistema Nervioso Cívico de Chile  
* **Misión:** Transformar frustración ciudadana en acción democrática concreta  
* **Visión:** Ser el sistema nervioso que conecta ciudadanía con instituciones  
* **Principio Rector:** Completitud \> Velocidad (F-04)

## **1.2 Propuesta de Valor**

CIVICUM ofrece una plataforma única que:

* Integra 43+ fuentes de datos cívicos oficiales chilenos  
* Opera con Honestidad Radical (F-02): sin letra chica ni promesas vacías  
* Garantiza Apartidismo Total (F-05): neutralidad política verificable  
* Funciona offline con dignidad (F-07): accesible sin conexión  
* Protege privacidad con sistema 1 persona \= 1 voto

# **4\. ARQUITECTURA TÉCNICA**

## **4.1 Stack Zero-Cost (F-43)**

Infraestructura de $0/mes perpetuo bajo límites documentados:

| Servicio | Límite Free Tier | Uso CIVICUM |
| :---- | :---- | :---- |
| Neon.tech | 10GB PostgreSQL | Base de datos principal |
| Vercel | 100GB bandwidth | Hosting frontend \+ Edge Functions |
| Cloudflare | CDN \+ WAF \+ R2 (10GB) | Cache \+ Seguridad \+ Storage |
| GitHub Actions | Ilimitado repos públicos | CI/CD completo |
| Workers AI | 10k tokens/día | Fallback IA (solo emergencias) |

## **4.2 Arquitectura IA Distribuida 3 Pisos (F-44)**

### **Piso 0 \- Sin LLM (Plantillas \+ Reglas)**

* 50+ plantillas legales/administrativas pre-cacheadas  
* Formularios guiados paso a paso  
* FAQs con respuestas pre-generadas

### **Piso 1 \- IA Local (WebLLM \+ TensorFlow.js)**

* WebLLM: Llama 3 8B cuantizado (\~200MB)  
* TensorFlow.js: Moderación de toxicidad local  
* Procesamiento 100% en dispositivo del usuario

### **Piso 2 \- IA en Nube (Cloudflare Workers AI)**

* Fallback para dispositivos sin WebGPU  
* Tareas complejas puntuales con control de cuota  
* Caché semántico para reducir llamadas

## **4.3 Salvaguardas Anti-Alucinación (F-17)**

* **Política Evidence-Only:** Solo respuestas con fuentes citadas  
* **Abstención Dura:** "No tengo datos verificados para responder"  
* **Verificador NLI:** Coherencia entre respuesta y fuentes  
* **Nunca inventa:** números, leyes, artículos o citas  
* **Disclaimers legales:** obligatorios en toda asesoría

# **5\. ARQUITECTURA OFFLINE-FIRST (F-07, F-33)**

## **5.1 Tres Experiencias según Conectividad**

| Tier | Caché | Funciones | IA |
| :---- | :---- | :---- | :---- |
| HIGH (Completa) | \~15MB | Todas \+ datos tiempo real | WebLLM activo |
| MEDIUM (Ligera) | \~3MB | Core \+ datos cacheados | Plantillas \+ reglas |
| LOW (Resiliencia) | \~800KB | Básicas \+ formularios | Sin IA, sync diferido |

## **5.2 Tecnologías Implementadas**

* Service Workers con estrategias Workbox  
* IndexedDB para almacenamiento local  
* Background Sync API para sincronización diferida  
* Progressive Enhancement para degradación elegante
