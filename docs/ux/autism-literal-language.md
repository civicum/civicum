# Lenguaje Literal en UI (Espectro AutistaSquad 212)

**Misión:** Revisión de literalidad. ¿Las metáforas confunden? Claridad absoluta.

## 1. El Riesgo del Lenguaje Figurativo
El diseño UX "Joven y Cool" (Tipo Startup o Red Social) usa frases proverbiales:
- *"¡Ponte la camiseta por tu barrio!"*
- *"Rompe el hielo en esta Mesa de Trabajo"*
- *"Súbete al carro de esta iniciativa"*

Para usuarios en el Espectro Autista (TEA), cuyo procesamiento cognitivo suele ser fuertemente **literal**, estas instrucciones generan alienación semántica. El usuario puede inferir que debe comprar ropa deportiva o que la iniciativa trata sobre transporte público. 

## 2. Pautaje de Redacción Cívica (Plain Literal Language)
El Rol 13 (Chilenización) y el Rol 07 (UX Writer) deben someterse a la regla neuro-inclusiva transversal:

1.  **Doble Test de Civia (NLP Audit):** Toda pantalla de interfaz o mensaje de Push Notification pasa por una pipeline automatizada en CI/CD donde el modelo LLM evalúa la ambigüedad. Si la oración depende de conocimiento idiosincrático no literal, es rechazada (Warning: `AmbiguityDetected`).
2.  **Reemplazos Oficiales (Glosario):**
    *   ❌ *"Ponte la camiseta"* -> ✅ *"Apoya este proyecto con tu firma"*
    *   ❌ *"Rompe el hielo"* -> ✅ *"Escribe el primer comentario"*
    *   ❌ *"Tu voto pesa oro"* -> ✅ *"Tu voto cuenta como 1 decisión validada"*
3.  **Eliminación del Sarcasmo Institucional:** CIVICUM jamás utilizará botones de salida con Dark Patterns sarcásticos ("No gracias, prefiero no ayudar a mi comuna").
